Reward Engine
Tokens accumulate per reading based on PM2.5 levels. Auto-syncs to Solana at 10 tokens.
PM2.5 Range (µg/m³) Air Quality Reward / Reading Reason
pm25 < 50 GOOD / MODERATE +0.02 Clean air bonus
50 ≤ pm25 < 100 UNHEALTHY SENSITIVE +0.01 Moderate reading
100 ≤ pm25 ≤ 300 VERY UNHEALTHY +0.005 Polluted but reporting
pm25 > 300 HAZARDOUS 0 Sensor fault suspected
When reward ≥ 10 and syncing === false → backend fires syncToSolanaAsync() (fire-and-forget) → calls addReward on-chain → resets local counter to 0. The syncing flag prevents concurrent sync attempts.
Solana Program
Program ID: 5ygRCA7pF2h7GeGxP9RaiNQNTNb5J5GnB9XSzxh75gVw
Instruction Signer What it does
initNode authority (backend)
nodeAccount (new keypair)		 Creates NodeAccount on-chain. Backend pays rent. Owner stored but does NOT sign.
addReward authority (backend) Increments reward_balance (lamports). Checks caller === stored authority.
claimReward owner (user wallet) Transfers lamports from NodeAccount → owner. Requires user signature — must be called from frontend.
NodeAccount struct
#[account]
pub struct NodeAccount {
    pub owner:             Pubkey,  // user wallet — stored, not required to sign
    pub authority:         Pubkey,  // backend wallet — signs addReward
    pub device_public_key: Pubkey,  // ESP32 Ed25519 public key
    pub reward_balance:    u64,     // lamports accumulated
}
// space = 8 (discriminator) + 32 + 32 + 32 + 8 = 112 bytes
API Reference
All authenticated routes require Authorization: Bearer <token>
POST /auth/signup Register new user ▼
Creates a new user account with a Solana wallet address. Returns a JWT immediately — no separate login needed. The wallet address is embedded in every token and flows through to all authenticated requests.
Internal flow
1
Check email uniqueness — throw ConflictError if taken
2
Hash password with bcrypt (10 rounds)
3
Save User to MongoDB including wallet field
4
Sign JWT with { userId, email, role, wallet } — 1 year expiry
5
Return user object + token

  "fullName": "Aether Node Owner",

  "password": "SecurePass123!",
  "wallet":   "44dZCzJ3nevs1KEYFwDgzDXTbqCnPBkxaeTK5RaA47gy"
}
Response

  "success": true,

Authenticates an existing user. Always re-login after schema changes to get a fresh token that includes all current fields. Old tokens missing wallet will cause req.user.wallet = undefined downstream.

Find user by email — NotFoundError if missing
2
bcrypt.compare password against stored hash
3
Sign JWT including wallet: user.wallet

  "password": "SecurePass123!"

No auth required Save this token
POST /node/create Register device off-chain ▼
Registers the ESP32 device in MongoDB. Setup step only — does not touch Solana. The devicePublicKey must be the real Ed25519 public key generated on the device. After this the device exists in the system but cannot send data yet (isLinked: false).
Internal flow
1
Check nodeId not already registered
2
Create Node document in MongoDB — isLinked: false, nodeAccount: null
3
Create NodeLatest placeholder with all sensors at 0
Request body
{
  "nodeId":          "NODE_001",
  "devicePublicKey": "4eHdNksHQ1dFspgWVxkS5japDtT8nvSySFJhNQz27GDz",
  "ownerEmail":      "owner@breezo.io",
  "ownerWallet":     "44dZCzJ3nevs1KEYFwDgzDXTbqCnPBkxaeTK5RaA47gy"
}
Requires JWT MongoDB only No Solana yet
POST /node/link/request Get signing challenge ▼
Step 1 of device linking. Server generates a random 32-byte hex challenge and stores it. The caller must sign it with the device's Ed25519 private key and return the signature in link/verify. This proves physical possession of the device.
Internal flow
1
Find device by devicePublicKey
2
Generate crypto.randomBytes(32).toString('hex')
3
Save to Node.linkChallenge in MongoDB
4
Return { devicePublicKey, challenge }
Request body
{
  "devicePublicKey": "4eHdNksHQ1dFspgWVxkS5japDtT8nvSySFJhNQz27GDz"
}
Response — save the challenge
{
  "success": true,
  "data": {
    "devicePublicKey": "4eHdNksHQ1dFspgWVxkS5japDtT8nvSySFJhNQz27GDz",
    "challenge":       "a3f9c12e847b3d1f...64-char hex...91f0"
  }
}
Requires JWT Save the challenge Sign it with device key
POST /node/link/verify Verify + create on-chain account ▼
Step 2 of device linking. Verifies the Ed25519 signature of the challenge. If valid, calls initNode on Solana to create a NodeAccount — backend wallet pays rent, user wallet stored as owner (does not sign). Marks node as linked in MongoDB.
Internal flow
1
Fetch node, verify linkChallenge exists
2
nacl.sign.detached.verify(challenge, signature, devicePublicKey)
3
Generate fresh Keypair for the on-chain NodeAccount
4
Call program.methods.initNode() — accounts: nodeAccount, owner, authority (backend, signer+payer), devicePublicKey, systemProgram
5
Save nodeAccount address to MongoDB, set isLinked: true
Request body
{
  "devicePublicKey": "4eHdNksHQ1dFspgWVxkS5japDtT8nvSySFJhNQz27GDz",
  "signature":       "3Vge8JU8Ux...base58 Ed25519 sig of challenge...fAf4"
}
Response
{
  "success": true,
  "data": {
    "nodeId":      "NODE_001",
    "isLinked":    true,
    "nodeAccount": "J9WpHHwftv7JFGcZjiDH4vXePcGV7ddL3K9YiMycww22"
  }
}
Requires JWT Hits Solana devnet wallet from JWT payload
POST /node/ingest Submit sensor reading (ESP32) ▼
Hot path — called by ESP32 every 15 seconds. No JWT required but the payload must be Ed25519-signed with the device's private key. Timestamp must be within 60 seconds (replay protection). Reward accumulates per reading and auto-syncs to Solana at 10 tokens.
⚠️ The signature must be the Ed25519 signature of the exact string JSON.stringify({ payload, timestamp }) — key order matters. The backend reconstructs this exact string to verify.
Internal flow
1
Find node by nodeId — reject if isLinked: false
2
Replay protection: Date.now() - timestamp > 60_000 → reject
3
Reconstruct message = JSON.stringify({ payload, timestamp })
4
nacl.sign.detached.verify(message, signature, devicePublicKey)
5
calculateReward(pm25) → upsert NodeLatest, accumulate reward
6
Async: write to SensorHistory collection for historical data
7
If reward ≥ 10 and !syncing → markSyncing() → fire syncToSolanaAsync()
Request body (no auth header needed)
{
  "nodeId":    "NODE_001",
  "timestamp": 1745123456789,       // Date.now() on device
  "signature": "4Tz9...base58 sig...Xm1",
  "payload": {
    "temperature": 28.5,
    "humidity":    62.3,
    "pm25":        42.0,
    "pm10":        67.8,
    "aqi":         112,
    "aqiLevel":    "MODERATE",
    "location":    { "lat": 28.6139, "lng": 77.2090 }
  }
}
No JWT — device route Ed25519 verified Replay protected (60s) Auto Solana sync

Returns all NodeLatest records for the authenticated user. This is the data source for a frontend dashboard — live sensor readings, AQI status, accumulated reward balance, and last seen timestamp for every node owned by the user.

POST /node/reward/claim Claim on-chain reward ▼
Returns the data needed to construct a claimReward transaction. Because claimReward requires the owner to sign and the owner's private key lives in the user's wallet (Phantom etc.), the backend returns the necessary accounts — the frontend must construct and send the transaction.
⚠️ The Rust ClaimReward struct has owner: Signer — meaning the user's wallet must sign. The backend cannot sign on behalf of a user. In production, this endpoint returns unsigned transaction data for the frontend to sign via Phantom/Backpack.
Request body
{
  "nodeId": "NODE_001"
}
Response (data for frontend to sign)
{
  "success":  true,
  "data": {
    "nodeAccount": "J9WpHHwftv7JFGcZjiDH4vXePcGV7ddL3K9YiMycww22",

    "programId":   "5ygRCA7pF2h7GeGxP9RaiNQNTNb5J5GnB9XSzxh75gVw"

Signature Helper Scripts
Run locally to generate valid signatures for testing.
scripts/sign-link-verify.ts
import nacl from "tweetnacl";
import bs58  from "bs58";

const DEVICE_PRIVATE_KEY = "2GZkTAT6eK3Upt...your 64-byte base58 key...JMjNAY";
const challenge          = "PASTE_CHALLENGE_HEX_FROM_LINK_REQUEST_HERE";

const keypair   = nacl.sign.keyPair.fromSecretKey(bs58.decode(DEVICE_PRIVATE_KEY));
const signature = nacl.sign.detached(
  new TextEncoder().encode(challenge), keypair.secretKey
);

console.log("devicePublicKey:", bs58.encode(keypair.publicKey));
console.log("signature:      ", bs58.encode(signature));
scripts/sign-ingest.ts
import nacl from "tweetnacl";
import bs58  from "bs58";

const DEVICE_PRIVATE_KEY = "2GZkTAT6eK3Upt...your key...JMjNAY";
const timestamp          = Date.now();
const payload = {
  temperature: 28.5, humidity: 62.3,
  pm25: 42.0,        pm10: 67.8,
  aqi: 112,           aqiLevel: "MODERATE",
  location: { lat: 28.6139, lng: 77.2090 }
};

const message   = JSON.stringify({ payload, timestamp });
const keypair   = nacl.sign.keyPair.fromSecretKey(bs58.decode(DEVICE_PRIVATE_KEY));
const signature = nacl.sign.detached(new TextEncoder().encode(message), keypair.secretKey);

console.log("timestamp:", timestamp);
console.log("signature:", bs58.encode(signature));
